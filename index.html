JDBC：

1. 概念：Java DataBase Connectivity  Java 数据库连接， Java语言操作数据库
    * JDBC本质：其实是官方（sun公司）定义的一套操作所有关系型数据库的规则，即接口。各个数据库厂商去实现这套接口，提供数据库驱动jar包。我们可以使用这套接口（JDBC）编程，真正执行的代码是驱动jar包中的实现类。

2. 快速入门：
    * 步骤：
        1. 导入驱动jar包 mysql-connector-java-5.1.37-bin.jar
            1.复制mysql-connector-java-5.1.37-bin.jar到项目的libs目录下
            2.右键-->Add As Library
        2. 注册驱动
        3. 获取数据库连接对象 Connection
        4. 定义sql
        5. 获取执行sql语句的对象 Statement
        6. 执行sql，接受返回结果
        7. 处理结果
        8. 释放资源

    * 代码实现：
          //1. 导入驱动jar包
        //2.注册驱动
        Class.forName("com.mysql.jdbc.Driver");

    public class JDBCUtils {
    private static String url;
    private static String user;
    private static String password;
    private static String driver;
    /**
     * 文件的读取，只需要读取一次即可拿到这些值。使用静态代码块
     */
    static{
        //读取资源文件，获取值。

        try {
            //1. 创建Properties集合类。
            Properties pro = new Properties();

            //获取src路径下的文件的方式--->ClassLoader 类加载器
            ClassLoader classLoader = JDBCUtils.class.getClassLoader();
            URL res  = classLoader.getResource("jdbc.properties");
            String path = res.getPath();
            System.out.println(path);///D:/IdeaProjects/itcast/out/production/day04_jdbc/jdbc.properties
            //2. 加载文件
           // pro.load(new FileReader("D:\\IdeaProjects\\itcast\\day04_jdbc\\src\\jdbc.properties"));
            pro.load(new FileReader(path));

            //3. 获取数据，赋值
            url = pro.getProperty("url");
            user = pro.getProperty("user");
            password = pro.getProperty("password");
            driver = pro.getProperty("driver");
            //4. 注册驱动
            Class.forName(driver);
        } catch (IOException e) {
            e.printStackTrace();
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        }
    }


    /**
     * 获取连接
     * @return 连接对象
     */
    public static Connection getConnection() throws SQLException {

        return DriverManager.getConnection(url, user, password);
    }

    /**
     * 释放资源
     * @param stmt
     * @param conn
     */
    public static void close(Statement stmt,Connection conn){
        if( stmt != null){
            try {
                stmt.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }

        if( conn != null){
            try {
                conn.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }


    /**
     * 释放资源
     * @param stmt
     * @param conn
     */
    public static void close(ResultSet rs,Statement stmt, Connection conn){
        if( rs != null){
            try {
                rs.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }

        if( stmt != null){
            try {
                stmt.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }

        if( conn != null){
            try {
                conn.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }

}

JDBC控制事务：

1. 事务：一个包含多个步骤的业务操作。如果这个业务操作被事务管理，则这多个步骤要么同时成功，要么同时失败。
2. 操作：
    1. 开启事务
    2. 提交事务
    3. 回滚事务
3. 使用Connection对象来管理事务
    * 开启事务：setAutoCommit(boolean autoCommit) ：调用该方法设置参数为false，即开启事务
        * 在执行sql之前开启事务
    * 提交事务：commit() 
        * 当所有sql都执行完提交事务
    * 回滚事务：rollback() 
        * 在catch中回滚事务

4. 代码：
    public class JDBCDemo10 {

        public static void main(String[] args) {
            Connection conn = null;
            PreparedStatement pstmt1 = null;
            PreparedStatement pstmt2 = null;

            try {
                //1.获取连接
                conn = JDBCUtils.getConnection();
                //开启事务
                conn.setAutoCommit(false);

                //2.定义sql
                //2.1 张三 - 500
                String sql1 = "update account set balance = balance - ? where id = ?";
                //2.2 李四 + 500

                //3.获取执行sql对象
                pstmt1 = conn.prepareStatement(sql1);
                pstmt2 = conn.prepareStatement(sql2);
                //4. 设置参数

                pstmt1.setInt(2,1);

                // 手动制造异常

                    }

                    e1.printStackTrace();

                JDBCUtils.close(pstmt1,conn);

            }

事务：

事务的介绍 ：

一个有多个步骤业务操作，被事务管理，那么这些操作要么全部成功，要么全部失败

操作：

1.开启事务：start transaction – 开启事务

2.回滚：

– 发现问题 开始回滚事务
ROLLBACK;

3 提交：

– 发现事务没有问题 提交事务
COMMIT;

– 添加事务
– 将两个操作变成一个整体
START TRANSACTION – 开启事务

UPDATE account SET balance = balance - 500 WHERE NAME = ‘zhangsan’;
错误。。。

UPDATE account SET balance = balance + 500 WHERE NAME = ‘lisi’;
– 发现事务没有问题 提交事务
COMMIT;

– 发现问题 开始回滚事务
ROLLBACK;

事务的提交方式：

自动提交：每执行一次sql语句就进行一次提交

手动提交：需要手动开启事务 start transaction

查看当前事务的提交方式 SELECT @@autocommit;查看事务的状态 1是自动提交 0

是 手动提交

更改i提交方式：SET @@autocommit；

事务的四大特征：

原子性：是不可在分割的最小操作单位，要么同时成功
持久性：事务提交或回滚后，数据库会持久化保存数据
隔离性：多个事务之间，相互独立。
一致性：事务操作前后事务总量不变
事务级别的隔离示范

如果多个事务操作同一批数据，则会引发一些问题，设置不同的隔离级别来解决；

存在问题

脏读：一个事务，读取到另一个事务没有提交的数据

不可重复读（虚读）：在同一个事务中。两次读到的事务不一样。

幻读：一个事务操作（DML）数据表中的所有记录，另一个事务操作了一条记录，则第一个事务查询不到自己的修改

隔离级别：

1.read uncommited:读未提交

*产生问题：脏读 不可重复读 幻读

2.read committed:读未提交

*产生问题：不可重复读 幻读

3.repeatable committed 可重复读

*产生问题：幻读

4 serializable:串行化

可解决所有问题

隔离等级越高 越安全，但效率越低

select @@tx_isolation;_

Annotation(注解)

定义

注解又名元数据

相似的定义：

注释：用来给程序员看本段代码的功能

注解：给计算机看，说明程序

功能：

1 编写Javadoc文档，通过注解名称生成对应的javadoc（API）文档

2 跟踪代码依赖性：实现替代配置文件功能。比如Dagger 2 依赖注入，未来java 开发，将大量注解配置，具有很大用处;

3 进行代码检查：比如override就是检查你这个方法有没有继承父类

初级阶段常用注解

override：override就是检查你这个方法有没有继承父类

Deprecated： 标记类型注解 放在被标记方法前，表示该方法有更好的解决方案，但是依旧可以正常使用

SuppressWarnings：警告压制 放在方法前 取消该方法的警告 放在类前 取消整个类的警告

（自定义）格式和本质

内置注解：@注解名

自定义注解：

public @interface 注解名

本质：自定义注解继承了一个特殊的接口（因为和接口的定义方法相同）

public interface 注解名 extends java.lang.annotation.Annotation{

//实际上继承了annotation接口

}

自定义注解
可返回属性：

1 基本数据类型

2 String

3 枚举类型(Enum)

4 注解

5 以上类型的数组

没有void

属性的定义：

1 在使用注解时要给注解的属性赋值，也可以在定义属性时加上default，后面直接加上属性值

例：@Test(age = 1)

2 如果注解的只有一个属性且名称为value,则在使用时，就不用写（属性=？）来赋值，可以直接写属性值

例：@Test(1)

3 数组赋值时要用大括号包括

例：@Test(a={“as”,”saf”})

Junit单元测试：

Junit是软件编写中用于单元测试的模块

测试分为：

1：白盒测试–测试时会看到代码的运行逻辑执行效果和效率，难度较高

2：黑盒测试–测试时不会关注代码的具体运行，只关注代码的运行结果是否符合预期

Junit是属于白盒测试的一种

Junit代码规范：

定义包

1.定义一个测试用例（不重要）

2.定义一个测试包（与被测试类的包分开），格式：域名.包名.Test（njcit.erp.test）

定义测试类

1.定义一个测试类：被测试类+Test（studentTest）

2 导入Junit依赖

3 定义测试方法，再方法前添加@Test 使得测试方法，可以单独运行

4 返回值为void（因为仅仅是为了测试，不提供其他功能）

结果判定：

1 看控制台是否为绿色（成功）/红色（失败）

2 因为有时候输出结果不一定准确，所以需要使用 断言 这一方法 译为：输入一个期盼值和一个程序结果去比对 看控制台颜色是否为绿色；

其他

@before（初始化方法） 在所以测试方法调用之前，先调用这一方法

@after （资源释放方法）在所以测试方法调用之后，必将调用此方法（即便出现异常）
