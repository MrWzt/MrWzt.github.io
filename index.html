SQL语言：

定义：

sql语言是一个关系型数据库操作的标准语言，它规定了所以关系型数据库的操作规则

sql语言的分类

  • DDL（Data Definition language ）数据定义语言–用来操作–数据库和表

  • DML（Data Manipulation Lauguage）数据操作语言–用来对数据库中的数据进行增删改

  • DQL（Data Query Lauguage）数据查询语言 用来查询数据表中的记录

  • DCL（Data Control Language）用来定义数据库的访问权限和安全级

约束

对表中的数据进行限定，保证数据的有效性，完整性，正确性

分类：

1 主键约束：primary key（

1  非空且唯一     2一张表只能有一个   3 主键是表中的唯一标识

创建表时添加主键

CREATE TABLE stu{

id INT PRIMARY KEY，—给id添加为主键

NAME VARCHAR NOT NULL–添加非空约束

}

删除主键

ALTER TABLE stu DROP PRIMARY KEY:

2 非空约束：not null

创建表时给列加非空约束

CREATE TABLE stu{

id INT;

NAME VARCHAR NOT NULL--添加非空约束

}

–删除name的非空约束

ALTER TABLE stu MODIFY NAME VARCHAR(20)

–创建后添加非空约束

ALTER TABLE stu MODIFY NAME VARCHAR(20) notnull;

3 外键约束：foreign key

4** 唯一约束：unique**

被修饰的那一列的值不能重复的

CREATE TABLE stu{

id INT;

NAME VARCHAR(20) unique;–添加了唯一约束

–删除唯一约束

ALTER TABLE stu DROP INDEX NAME;

创建表后添加唯一约束

ALTER TABLE stu modify NAME varchar(20) unique

注意：此时添加唯一约束时要确定 表中没有重复的数值

}

多表之间的关系

一对多关系：

在多的一方建立外键，指向一的一方的主键。

多对多关系：

需要借助第三章中间表 中间表至少包含两个字段，这两个字段为外键，分别指向两张的主键

一对一关系实现：

在任意一方添加唯一外键指向另一方的主键

范式：数据库设计规范：将表不断划分进行解耦，减少表中数据的依赖

第一范式问题：

1.数据存在大量冗余（第二范式已解决）

2.书局添加存在问题：添加新的字段，数据可能不合法（三式已解决）

3 数据存在删除问题： 删除一个字段 其他字段也会被删除；（第三范式已解决）

第二范式存在问题：

函数依赖：A–>B

如果可以通过A属性确定唯一的B属性 那么就说B依赖于A；

也可以通过两个属性组成的属性组确定一个唯一的分数

完全函数依赖：A–>B

如果A是一个属性组，那么B依赖于A中的所有属性；

不完全函数依赖：A–>B

如果A是一个属性组，那么B部分依赖于A中的属性

传递函数依赖 A–>B ,B–>
A确定B ，B确定C 那么称C传递依赖于A

码：在一张表中通过一个唯一的属性或属性组，被其他属性所完全依赖

该属性或属性组，被称为码；

主属性：码中的所有属性

非主属性：码以外的所有属性

Annotation(注解)

定义

注解又名元数据

相似的定义：

注释：用来给程序员看本段代码的功能

注解：给计算机看，说明程序

功能：

1 编写Javadoc文档，通过注解名称生成对应的javadoc（API）文档

2 跟踪代码依赖性：实现替代配置文件功能。比如Dagger 2 依赖注入，未来java 开发，将大量注解配置，具有很大用处;

3 进行代码检查：比如override就是检查你这个方法有没有继承父类

初级阶段常用注解

override：override就是检查你这个方法有没有继承父类

Deprecated： 标记类型注解 放在被标记方法前，表示该方法有更好的解决方案，但是依旧可以正常使用

SuppressWarnings：警告压制 放在方法前 取消该方法的警告 放在类前 取消整个类的警告

（自定义）格式和本质

内置注解：@注解名

自定义注解：

public @interface 注解名

本质：自定义注解继承了一个特殊的接口（因为和接口的定义方法相同）

public interface 注解名 extends java.lang.annotation.Annotation{

//实际上继承了annotation接口

}

自定义注解
可返回属性：

1 基本数据类型

2 String

3 枚举类型(Enum)

4 注解

5 以上类型的数组

没有void

属性的定义：

1 在使用注解时要给注解的属性赋值，也可以在定义属性时加上default，后面直接加上属性值

例：@Test(age = 1)

2 如果注解的只有一个属性且名称为value,则在使用时，就不用写（属性=？）来赋值，可以直接写属性值

例：@Test(1)

3 数组赋值时要用大括号包括

例：@Test(a={“as”,”saf”})

Junit单元测试：

Junit是软件编写中用于单元测试的模块

测试分为：

1：白盒测试–测试时会看到代码的运行逻辑执行效果和效率，难度较高

2：黑盒测试–测试时不会关注代码的具体运行，只关注代码的运行结果是否符合预期

Junit是属于白盒测试的一种

Junit代码规范：

定义包

1.定义一个测试用例（不重要）

2.定义一个测试包（与被测试类的包分开），格式：域名.包名.Test（njcit.erp.test）

定义测试类

1.定义一个测试类：被测试类+Test（studentTest）

2 导入Junit依赖

3 定义测试方法，再方法前添加@Test 使得测试方法，可以单独运行

4 返回值为void（因为仅仅是为了测试，不提供其他功能）

结果判定：

1 看控制台是否为绿色（成功）/红色（失败）

2 因为有时候输出结果不一定准确，所以需要使用 断言 这一方法 译为：输入一个期盼值和一个程序结果去比对 看控制台颜色是否为绿色；

其他

@before（初始化方法） 在所以测试方法调用之前，先调用这一方法

@after （资源释放方法）在所以测试方法调用之后，必将调用此方法（即便出现异常）
